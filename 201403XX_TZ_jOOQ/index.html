Les principes

SQL au premier plan

  • Database first: Schema base → Code Java
  • Monde relationnel au naturel:
    • SQL au premier plan
    • des tuples/enregistrement (Record) plutôt que des objets
  • Seulement du relationnel:
    • Pas des relations 1-N ou N-N, pas d'héritage
    • Pas de requêtage sur des collections (QueryDSL, LinQ, Slick)
    • Pas de transactions, pas de cache

Tout le SQL

SQL Harmonisé

  • SQLDialect, comme Hibernate
  • Exemples:
    • merge / insert ... on duplicate key update
    • limit ... offset / rownum
    • nvl / ifnull

Java: API Fluent SQL

En Java 

Personne p=PERSONNE.as("p");
select(p.ID, p.NOM,p.PRENOM)
    .from(p)
    .where(p.NOM.like("Q%"));
En SQL 

select p.ID, p.NOM, p.PRENOM
    from PERSONNE as p
    where p.NOM like 'Q%'

Java: Type Safe

Active Records

  • Pour les CRUDs, comme Rails
  • Typesafe, contrairement à Rails

PersonneRecord personne = jooq
    .selectFrom(PERSONNE)
    .where(PERSONNE.ID.eq(id)).fetchOne();
personne.setPrenom("George");
personne.store();

  4. Eventuellement, modifier un enregistrement
